# /goal テンプレ

`/goal` は機械判定で進めるための作業単位。必ず以下を明示する:
- **達成状態**: 状態 or 数値で書く(例「`npm run build` が exit 0」「検索が出典付きで3件返す」)。曖昧語は禁止。
- **対象設計**: docs/design/ の該当書(design-review で全レンズ PASS 済み)。
- **禁止事項**: 触ってはいけないパス/操作(元 repo・.env・破壊的 SQL 等)。
- **ターン上限**: 上限を決め、超えたら停止して報告する。
- **節目 commit**: サブゴール達成ごとに commit。

完了判定は acceptance-judge / Stop hook が**独立検証**する(作業役と判定役を分離)。
